> “工具的意义，不在于它多么智能，而在于它能否帮你更清晰地认识自己。”

## 一、从碎片到体系

过去几年，我积累了大量的笔记：工作中的代码片段、读书摘录、旅行计划、灵感想法……  
它们散落在各个地方：Apple Notes、浏览器收藏夹、GitHub issue、甚至聊天记录。  
每次想找点东西，像是在大海捞针。

直到我认真搭建了自己的知识体系：  
**Apple Notes → MCP Agent → Obsidian → 博客输出。**

---

## 二、Apple Notes：我的输入端

Apple Notes 是我最自然的输入工具。  
它启动快、跨设备同步、支持语音和图片。  
我把它当作「灵感收集盒」，所有临时想法、会议速记、读后感都先放这里。

这部分信息是不成体系的。  
关键在于，**要让它自动流入到系统的中心——Obsidian。**

---

## 三、自动化同步：MCP + 定时任务 + Agent

每天凌晨，我的一个小 Agent 会自动执行一次同步任务：

1. 使用 **Apple MCP** 读取新的 Notes；
2. 通过 **LangGraph workflow** 调度；
3. 调用 **Obsidian MCP API** 将内容写入我的知识库。

同步过程是全自动的，不需要手动导入。  
它不仅保留了原始文本，还会自动在 YAML 头部添加元信息（时间戳、标签、来源）。

> “人的思维是流动的，知识的体系却要稳定。”  
>  自动化同步，就是让这两者找到平衡。

---

## 四、Obsidian：知识的中枢

我用 Obsidian 来承载所有正式笔记、学习记录和项目文档。  
以下是我目前安装的主要插件：

- **Admonition**：让笔记更有层次感，支持提示框、警告框等；
- **Attachment Management**：自动整理附件，防止资源散乱；
- **Find Orphaned Files and Broken Links**：清理未引用的文件；
- **Git**：每天自动提交版本；
- **Local REST API**：开放接口给 MCP agent 调用；
- **Smart Chat** 与 **Smart Connections**：AI 驱动的智能关联；
- **Templater**：模板化笔记，生成结构化日志与学习计划。

我为不同主题建立了主文件夹：
- `/Tech`：编程、项目笔记；
- `/Life`：日常、旅行；
- `/Reading`：读书笔记；
- `/Ideas`：想法与随笔。

通过 **Smart Connections** 插件，笔记之间会自动建立语义链接。  
就像在我的知识库里，思想会自己“长出神经元”。

---

## 五、Folo + Web Clipper：输入的另一条路

除了 Apple Notes，我还用 **Folo** 插件和浏览器的 **Web Clipper** 把看到的网页文章直接保存到 Obsidian。  
互联网的内容碎片化、易失。  
统一保存、统一备份，是为了让知识“沉淀”下来。

这些网页笔记会自动归类到 `/Inbox`，等待整理和归档。

---

## 六、输出：从笔记到博客

我把 Obsidian 作为所有内容的源头（Single Source of Truth）。  
整理完的内容通过 **Pandoc Plugin** 转为 Markdown 或 MDX，发布到我的博客。

有些是技术总结；
有些是生活随笔；
但它们都有一个共同点：  
**来源于真实的积累。**

> “输出是最好的输入。”  
> 每一次写博客，都是一次自我认知的更新。

---

## 七、总结

整个体系可以用一句话概括：

> Apple Notes 捕捉 → MCP 同步 → Obsidian 整理 → 博客输出。

![](https://raw.githubusercontent.com/yourname/blog-assets/main/obsidian-system-diagram.png)

这套系统让我逐渐从「收集信息的人」变成「生产知识的人」。

如果你也在寻找属于自己的数字脑，  
不妨从一个简单的同步任务开始。
